HTML

HTML stands for Hypertext Markup Language. HTML documents are what you see on your browser. They can have text content which describes ""what you see" and markup which describes "how it looks". This Video is a good overview of this.

Tags and Elements

HTML documents contain HTML elements and they tell the browser which element to use by using tags. We use tags to make things bold or to add italics. We can also use tags to insert breaks or to format the page on how we want it to look. HTML 5.0 is the language that the web is programmed in today.

Inline VS Block Elements

HTML Elements can be Inline or Block. Block elements form invisible boxes around the data inside of them. Inline does not.

Lesson 2 Notes

Basics of the Web and HTML

The web is a series of HTML Documents. We use URL's to access websites today or we can use hyperlinks.

HTML and Tags

HTML Markup is made up of tags and each tag has what are known as attributes. For example in a tag attribute- the "a" anchor tag, the attribute is the href point. In a image tag, its the src link.
"img" tags are void tags since they dont actually contain content.

White Space and Block Elements

HTML documents contain white space. In order to break up the white space, you can use a void tag like break or a paragraph tag. A break tag is inline, all it does is end a line.

A paragraph tag is a block element- it makes an invisible box around what ever is inside the tags. We will later learn that all webpages are just a series of boxes.

Span and Div are other examples of block and inline elements. Span is inline, div is block. CSS can be added to a span and div element to add style to HTML

a, span, br, img, and strong are all inline elements. p and form are block elements.

Complete HTML Documents, Elements and DOM

Complete HTML Documents must have an open, a head, a body and a close.

All Elements are boxes that contain tree like structures. HTML and CSS are both programming languages. HTML controls the structure of a web page. CSS controls the style of the page.

When programmers talk about Document Object Model(DOM), they are talking about the tree like structure of a web page, as every page can have multiple elements embedded with styles and other things in them.

Avoid Repetition

Avoiding repetion is important because it helps you avoid errors. If you dont avoid repetition, you will have to do the same thing over and over again as you write your code and that can lead to errors via manual typing or copy/pasting.

It will also help with consistency because if you wanted things to look the same, it allows you to give a each div the same class and then you can specify that look in just one place.

CSS Codes

All documents should start with a line telling the browser what kind of doctype it is. Usually they are HTML files.
You should indent when you program so its easier to read your code.

How CSS Works and How You Use it

You can add style without linking to a seperate file by using the style tag.

Keeping the link seperate keeps the code and style on seperate pages so that it makes it easier to understand and see in web developer tools.

CSS stands for Cascading Style Sheets and it helps the programmers avoid repetion. The "C" for cascading is the most important factor for CSS.

CSS determines the style of the webpage you are looking at. It is used to control the look of the page.

CSS can have rules and refereneces for several style sheets.The analogy is to think of it as laws- Federal, State, Municipal and House Laws. In terms of which rule it will use, the most specific rule applies.

The rule of inheritance applies to CSS. It relies on a ancestor-descendent relationship to operate.

In CSS, you first must write a selector tag- this defines what element of a page the CSS will apply too.
You can select an HTML tag and that rule will apply to all tags in the sheet.

.p {} add style to all paragraphs from example. To add style to a description, you would type .description

How Do I Get More Specific with CSS

In CSS, you need a selector(ex. H1), and you need a rule.
In the body of that rule you need a "declaration" a "property" and a "value".

How To Include CSS Styling in my Own Web page

Method 1- Write CSS in the "head" of you HTML file. This is good for small project.

Method 2- Link your HTML to a seperate CSS file. As long as they are saved in the same folder, you style attributes should display with your code. This is better for larger projects as it seperates your style from you code.

Method 3- the least recommended is the Style Inline method where you write it inline with your code. This is considered a bad idea since it leads to a lot of repeated code.

The Box Revisited

CSS styling follows the BOX model. It contains four distinct pieces Margin, Border, Padding and Color

All HTML elements are boxes and they are a subset of 4 boxes whom I have labeled above and there are many ways to set the attributes for the boxes.

Box positions are also important and you need to understand the code that you write to see how they work. The DIV tag is a block element, so it sets a block around the content. Other tags set different boxes around the content as well

Following Patterns

Eventually, after we code, we will go thru a pattern of Code, Test and Refine.

We need to build structure into our HTML document and visualize boxes. By adding the correct tags, headers and CSS, our webpage will come to life.

In order to make sure we are efficient and effective in coding webpages from a design, we should be:

  1. Looking for Natural boxes
  2. Looking for repeated styles and semantic benefits
  3. Writing your HTML
  4. Apply CSS styles (starting from biggest to smallest)
  5. Test and refine
